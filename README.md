# Шерстяной Синдикат

Telegram-бот с игровой петлей про рыбов, мышей, рост по девяти жизням, PvP и чатовые события. Новый игрок начинает с базовой бригадой из 3 мышей, чтобы первые рискованные действия не упирались в пустой старт.

## Игровые системы

- `кузница` — броня и классовое оружие из шерсти, металла, мусора и небольшой оплаты рыбами.
- `магазин` — расходники за рыбов.
- `ежедневки` — дневные задания со стриком, растущей сложностью и наградами.
- `надеть название` — надеть шлем, нагрудник, нарукавники, ботинки или оружие своего класса.
- `экипировка` — посмотреть надетую экипировку.
- `использовать название` — использовать расходник и получить бафф на следующую тематическую команду.

Расходники v1: `Валерьянка`, `Кошачья мята`, `Мышиный энергетик`, `Лапомазь`. Каждый расходник хранится одним стеком до 9999 штук; если стек полон, магазин не списывает рыбов. Магазин показывает баланс, текущий стек, дневной лимит партии и доступный максимум покупки. Покупать можно пачками: `/buy Валерьянка 10`, `/buy Валерьянка max` или текстовыми фразами вроде `купить Валерьянка 10`, `отсыпь кошачья мята max`.
Экипировка v1: четыре слота брони и слот оружия, все в трёх тирах качества. Класс открывается при переходе на вторую жизнь и выбирается один раз: танк, вор, саппорт или убийца. Оружие v1 даёт отдельную линейку для каждого класса. Ковать классовое оружие может только игрок с совпадающим классом; без выбранного класса кузница отказывает до списания ресурсов. В каждой линейке хранится один предмет с прочностью: повторная ковка того же рецепта чинит вещь, а ковка следующего тира улучшает её без дублей.

Ковка теперь имеет исходы: базово 90% удачной ковки и 10% провала. Внутри этих исходов есть критический успех и критический провал: крит-успех сохраняет ресурсы, крит-провал спасает предмет, но требует двойную цену ресурсов. Оплата кузнецу рыбами списывается при любой попытке ковки и не удваивается. Если вещь уже целая, кузница не тратит ресурсы на лишний ремонт.

После `старт` бот показывает основную клавиатуру. Кузница, магазин, инвентарь и экипировка работают через кнопки: игроку не нужно вручную вводить технические id, названия расходников или названия экипировки. Инвентарь показывает ресурсы, расходники и надетые вещи; свободная экипировка не выводится, потому что предмет либо надет, либо исчезает при поломке. Магазин и кузница используют компактные списки с карточками: в каталоге видны товары/рецепты, а покупка, продажа, корзина и сборка открываются уже внутри выбранной карточки. Раздел оружия в кузнице сначала делится на классы: танк, вор, саппорт и убийца.

Большинство экранов можно открыть обычным сообщением: `старт`, `помощь`, `мой котёнок`, `досье`, `инвентарь`, `кузница`, `магазин`, `ежедневки`, `экипировка`, `мяу`, `охота`, `работа`, `подвал`, `расти`, `укусить`, `топ`, `босс`, `контейнер`, `событие`. Команды со слэшем оставлены как запасной путь.

Ежедневные задания растут от стрика: первые 3 дня лёгкие, дни 4-9 средние, с 10-го дня — вкусные задания и награды. На каждый новый стрик даётся 3 сейва: пропуск дня тратит один сейв и сохраняет серию, но не начисляет награду за пропущенный день. Когда сейвы кончились, следующий пропуск начинает новый стрик с новым набором из 3 сейвов.

Для работы и подвала можно указывать количество обычным текстом: `работа 5`, `подвал 4`. Мыши списываются сразу и возвращаются отдельным сообщением после смены, поэтому отправленная бригада временно не защищает игрока.

У подвала есть критический успех: добытчики возвращаются без потерь, приносят усиленную добычу и могут привести новых мышей-рекрутов. Завершение работы и подвала разделено: `завершить работу` проверяет смену, `завершить подвал` проверяет добычу.

Профиль можно удалить текстовыми алиасами `сброс`, `сбросить профиль`, `удалить профиль`, `начать заново` или запасной командой `/reset_me`.

В группах Telegram обычные фразы работают только если у бота отключён privacy mode. Если фразы не появляются в логах, Telegram их не доставляет боту. Отключается через BotFather: `/setprivacy` → выбрать бота → `Disable`.

## Быстрый старт

1. Установи Python 3.11+.
2. Создай свежее окружение:

```powershell
py -3 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Если `py` не установлен, используй полный путь к своему `python.exe`:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

3. Подними PostgreSQL:

```powershell
docker compose up -d db
```

4. Создай `.env`:

```text
BOT_TOKEN=telegram_bot_token
DATABASE_URL=postgresql://postgres:cats_secret_password@localhost:5432/catsyndicate
```

Можно начать с копии шаблона:

```powershell
Copy-Item .env.example .env
```

После этого замени `put_telegram_bot_token_here` в `.env` на реальный токен от BotFather.

5. Запусти бота:

```powershell
.\.venv\Scripts\python.exe main.py
```

`.venv` не хранится в git и может ломаться при переносе проекта между пользователями или машинами. Если окружение ссылается на несуществующий Python, проще удалить папку `.venv` и создать её заново командами выше.

## Удобный запуск

Открыть проект в VS Code и сразу прогнать быстрые проверки:

```powershell
.\scripts\dev.cmd
```

Только проверки, без открытия VS Code:

```powershell
.\scripts\dev.cmd -NoCode
```

Открыть VS Code и запустить бота после проверок:

```powershell
.\scripts\dev.cmd -Bot
```

В VS Code также доступны задачи: `Catsyndicate: check all`, `Catsyndicate: test`, `Catsyndicate: compile`, `Catsyndicate: run bot`.

## Текущая шлифовка

- Общие проверки пользователя и cooldown вынесены в `services/game_utils.py`, чтобы хендлеры не дублировали одну и ту же обвязку.
- Callback-кнопки кузницы, инвентаря и экипировки используют общий guard для незарегистрированных игроков.
- Добавление ресурсов в инвентарь централизовано внутри `database/db_manager.py`, а старые неиспользуемые DB-методы удалены.
- Экраны экипировки и инвентаря форматируются отдельными helper-функциями; названия предметов экранируются перед HTML-выводом.
- Кузница и магазин разделены: экипировка куется за материалы и рыбов, расходники покупаются за рыбов.
- Надевание экипировки и использование расходников проходят через общие helper-функции в `handlers/crafting.py`, чтобы текстовые команды и callback-кнопки не расходились в поведении.
- Точка входа `main.py` использует обычный `logging` для сообщений запуска и остановки бота.
- Экипировка стала уникальной по линейке слота и получила прочность: повторная ковка того же предмета чинит его, следующий тир улучшает, а инвентарь показывает прочность вместо количества.
- Добавлены смешные текстовые алиасы вроде `молотком по судьбе`, `мыши на завод`, `экспедиция под диван` и `кусательный протокол`.
- Расходники ограничены одним стеком до 9999 штук; покупка сверх лимита отклоняется без списания рыбов и отвечает одной из 15 комичных реплик магазина.
- Магазин поддерживает покупку пачками `x1/x5/x10/x25/x100/max`, показывает количество, цену за штуку, итоговую сумму и баланс до/после покупки.
- Для покупки добавлены текстовые алиасы-префиксы вроде `купить`, `отсыпь`, `касса пробей`, `лавочник дай`; нехватка рыбов отвечает одной из 15 отдельных абсурдных фраз.
- Крупные покупки `x25`, `x100` и `max` идут через подтверждение; после успешной покупки появляется кнопка повтора той же пачки.
- В магазине есть вкладки `Все`, `Рост`, `Охота`, `Работа`, `Рыбовы`; каждый день случайно выбирается один товар дня со скидкой 20% и один товар для акции `3 по цене 2`. Каталог магазина показывает компактные строки, а действия `x1/x5/x10/x25/x100/max`, корзина и продажа живут в карточке товара. Также есть дневной лимит партии, корзина для нескольких товаров и продажа расходников обратно за 60% цены.
- Экономика магазина вынесена в `services/shop.py`, а дневные лимиты, корзина и журнал покупок/продаж хранятся в БД. Покупка расходника проходит одной транзакцией: проверка стака, баланса, дневного лимита, списание рыбов, выдача предмета и запись лога не расходятся при параллельных нажатиях.
- Корзина показывает баланс, итог и остаток после оплаты, умеет менять количество и удалять позиции. Если покупка не проходит, касса отдельно показывает нехватку рыбов, полный стек или упёршийся дневной лимит.
- История магазина показывает последние операции, суммарно потраченных/вернувшихся рыбов и любимый товар. Каждый день дополнительно выбирается товар с большой партией: его дневной лимит выше на 80 штук.
- Репутация у лавочника растёт от потраченных рыбов и даёт мягкий бонус к дневным лимитам. Отказы магазина не штрафуют игрока, но пишутся как подозрительные попытки ради комедийных ответов.
- Если покупка сорвалась из-за рыбов, лимита или полного стека, товар автоматически попадает в хотелки. Экран хотелок открывается кнопкой или фразами вроде `хотелки`, `список желаний`, `предзаказ`; он показывает, что уже можно купить, а что всё ещё упирается в причину отказа. Готовую позицию можно сразу купить, переложить в корзину или удалить из списка.
- “Полка дня” добавляет ежедневный мягкий модификатор для одного случайного товара: небольшую скидку/наценку или дополнительный лимит, с отдельной подписью в магазине.
