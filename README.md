# Шерстяной Синдикат

Telegram-бот с игровой петлей про рыбов, мышей, рост по девяти жизням, PvP и чатовые события. Новый игрок начинает с базовой бригадой из 3 мышей, чтобы первые рискованные действия не упирались в пустой старт.

## Игровые системы

- `кузница` — броня и классовое оружие из шерсти, металла, мусора и небольшой оплаты рыбами.
- `магазин` — расходники за рыбов.
- `ежедневки` — дневные задания со стриком, растущей сложностью и наградами.
- `надеть название` — надеть шлем, нагрудник, нарукавники, ботинки или оружие своего класса.
- `экипировка` — посмотреть надетую экипировку.
- `использовать название` — использовать расходник и получить бафф на следующую тематическую команду.

Расходники v1: `Валерьянка`, `Кошачья мята`, `Мышиный энергетик`, `Лапомазь`.
Экипировка v1: четыре слота брони и слот оружия, все в трёх тирах качества. Оружие v1 даёт отдельную линейку для каждого класса: танк, вор, саппорт и убийца.

Ковка теперь имеет исходы: базово 90% удачной ковки и 10% провала. Внутри этих исходов есть критический успех и критический провал: крит-успех сохраняет ресурсы и для экипировки повышает качество до следующего тира; крит-провал спасает предмет, но требует двойную цену ресурсов. Оплата кузнецу рыбами списывается при любой попытке ковки и не удваивается.

После `старт` бот показывает основную клавиатуру. Кузница, магазин и инвентарь работают через кнопки: игроку не нужно вручную вводить технические id, названия расходников или названия экипировки.

Большинство экранов можно открыть обычным сообщением: `старт`, `помощь`, `мой котёнок`, `досье`, `инвентарь`, `кузница`, `магазин`, `ежедневки`, `экипировка`, `мяу`, `охота`, `работа`, `подвал`, `расти`, `укусить`, `топ`, `босс`, `контейнер`, `событие`. Команды со слэшем оставлены как запасной путь.

Ежедневные задания растут от стрика: первые 3 дня лёгкие, дни 4-9 средние, с 10-го дня — вкусные задания и награды. На каждый новый стрик даётся 3 сейва: пропуск дня тратит один сейв и сохраняет серию, но не начисляет награду за пропущенный день. Когда сейвы кончились, следующий пропуск начинает новый стрик с новым набором из 3 сейвов.

Для работы и подвала можно указывать количество обычным текстом: `работа 5`, `подвал 4`. Мыши списываются сразу и возвращаются отдельным сообщением после смены, поэтому отправленная бригада временно не защищает игрока.

У подвала есть критический успех: добытчики возвращаются без потерь, приносят усиленную добычу и могут привести новых мышей-рекрутов. Завершение работы и подвала разделено: `завершить работу` проверяет смену, `завершить подвал` проверяет добычу.

Профиль можно удалить текстовыми алиасами `сброс`, `сбросить профиль`, `удалить профиль`, `начать заново` или запасной командой `/reset_me`.

В группах Telegram обычные фразы работают только если у бота отключён privacy mode. Если фразы не появляются в логах, Telegram их не доставляет боту. Отключается через BotFather: `/setprivacy` → выбрать бота → `Disable`.

## Быстрый старт

1. Установи Python 3.11+.
2. Создай свежее окружение:

```powershell
py -3 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Если `py` не установлен, используй полный путь к своему `python.exe`:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

3. Подними PostgreSQL:

```powershell
docker compose up -d db
```

4. Создай `.env`:

```text
BOT_TOKEN=telegram_bot_token
DATABASE_URL=postgresql://postgres:cats_secret_password@localhost:5432/catsyndicate
```

Можно начать с копии шаблона:

```powershell
Copy-Item .env.example .env
```

После этого замени `put_telegram_bot_token_here` в `.env` на реальный токен от BotFather.

5. Запусти бота:

```powershell
.\.venv\Scripts\python.exe main.py
```

`.venv` не хранится в git и может ломаться при переносе проекта между пользователями или машинами. Если окружение ссылается на несуществующий Python, проще удалить папку `.venv` и создать её заново командами выше.

## Удобный запуск

Открыть проект в VS Code и сразу прогнать быстрые проверки:

```powershell
.\scripts\dev.cmd
```

Только проверки, без открытия VS Code:

```powershell
.\scripts\dev.cmd -NoCode
```

Открыть VS Code и запустить бота после проверок:

```powershell
.\scripts\dev.cmd -Bot
```

В VS Code также доступны задачи: `Catsyndicate: check all`, `Catsyndicate: test`, `Catsyndicate: compile`, `Catsyndicate: run bot`.

## Текущая шлифовка

- Общие проверки пользователя и cooldown вынесены в `services/game_utils.py`, чтобы хендлеры не дублировали одну и ту же обвязку.
- Callback-кнопки кузницы, инвентаря и экипировки используют общий guard для незарегистрированных игроков.
- Добавление ресурсов в инвентарь централизовано внутри `database/db_manager.py`, а старые неиспользуемые DB-методы удалены.
- Экраны экипировки и инвентаря форматируются отдельными helper-функциями; названия предметов экранируются перед HTML-выводом.
- Кузница и магазин разделены: экипировка куется за материалы и рыбов, расходники покупаются за рыбов.

## Проверки

Синтаксис проекта:

```powershell
.\.venv\Scripts\python.exe -m compileall main.py data database handlers services tests
```

Быстрые unit-тесты игровой логики, форматтеров и helper-ов:

```powershell
.\.venv\Scripts\python.exe -m unittest discover
```
